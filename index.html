The OSI Model Explained Simply

The OSI Model: everyone mentions it, few explain it well. Now I'm actually understanding it.

The 7 Layers (Bottom to Top)

Layer 1: Physical

Cables, signals, bits. Examples: Ethernet cables, Wi-Fi, fiber optics

Layer 2: Data Link

MAC addresses. Devices: Switches, bridges

Layer 3: Network

IP addresses, routing. Devices: Routers

Layer 4: Transport

TCP (reliable), UDP (fast). Ports: 80=HTTP, 443=HTTPS

Layer 5: Session

Managing connections between applications

Layer 6: Presentation

Data formatting, encryption, compression

Layer 7: Application

HTTP, FTP, SMTP, DNS, SSH

Why It Matters

Troubleshooting framework: No Wi-Fi? Layer 1. Can't reach other networks? Layer 3.

Subnetting Made Easy

Subnetting used to terrify me. Now I'm understanding it properly.

What Is Subnetting?

Dividing one large network into smaller logical networks.

Common Subnet Masks

CIDRSubnet MaskUsable Hosts
/24255.255.255.0254
/25255.255.255.128126
/26255.255.255.19262
/27255.255.255.22430
/30255.255.255.2522

The Formula

2^(host bits) - 2 = usable hosts

Subtract 2 for network and broadcast addresses.

My First GNS3 Lab

Time for hands-on practice. Building and configuring a network from scratch.

Lab Topology

PC1 --- R1 --- R2 --- PC2

Configure R1

Router> enable
Router# configure terminal
Router(config)# hostname R1
R1(config)# interface Gi0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1# copy run start

Add Static Routes

R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2

Hands-on practice beats theory every time.

Understanding VLANs

VLANs let you segment networks logically without buying more switches.

Basic VLAN Configuration

SW1(config)# vlan 10
SW1(config-vlan)# name SALES
SW1(config)# interface fa0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config)# interface fa0/24
SW1(config-if)# switchport mode trunk

Verification

show vlan brief
show interfaces trunk

Static vs Dynamic Routing

Understanding when to manually configure routes versus letting protocols handle it.

Static Routing

Manually configured routes. Simple, predictable, no overhead. Best for small networks.

ip route 192.168.2.0 255.255.255.0 10.0.0.2

Dynamic Routing

Routers automatically share routes. Protocols: RIP, OSPF, EIGRP. Best for large, changing networks.

Comparison

FactorStaticDynamic
ConfigurationManualAutomatic
Adapts to changesNoYes
ScalabilityPoorGood
