Subnetting Made Easy (I Hope)

Subnetting used to terrify me. I memorized tables and barely passed. Now I'm actually understanding it.

What Is Subnetting?

Dividing one large network into smaller logical networks. Like dividing an office building into departments.

IP Address Basics

IPv4 address: 192.168.1.100

Two parts:

  • Network portion: Which network
  • Host portion: Which device

The subnet mask tells you where to split.

Common Subnet Masks

CIDR Subnet Mask Usable Hosts
/24 255.255.255.0 254
/25 255.255.255.128 126
/26 255.255.255.192 62
/27 255.255.255.224 30
/30 255.255.255.252 2

The Formula

2^(host bits) - 2 = usable hosts

Subtract 2 because:

  • First address = network address
  • Last address = broadcast address

Example: 192.168.1.0/24

  • Network: 192.168.1.0
  • First usable: 192.168.1.1
  • Last usable: 192.168.1.254
  • Broadcast: 192.168.1.255
  • Total usable: 254 hosts

Practice Daily

What's working for me:

  • 10 subnet problems every morning
  • Test in GNS3 lab
  • Intentionally misconfigure and troubleshoot

It gets easier with repetition.

Scenario: User can't access website

Check:

  1. Layer 1: Cable plugged in? Wi-Fi connected?
  2. Layer 2: Switch port active?
  3. Layer 3: Can you ping the gateway?
  4. Layer 7: DNS working? Try IP directly

Subnetting Practice

Question: How many usable hosts in 10.0.0.0/26?

Answer:

  • /26 = 6 host bits (32-26)
  • 2^6 = 64 total
  • 64 - 2 = 62 usable hosts

My First GNS3 Lab: Basic Router Configuration

Time for hands-on practice. Building and configuring a network from scratch.

Lab Topology

PC1 --- R1 --- R2 --- PC2
    .1   .1 .2  .2 .1   .1

Step 1: Configure R1

Router> enable
Router# configure terminal
Router(config)# hostname R1

R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown

R1(config-if)# interface GigabitEthernet0/1
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# no shutdown

R1# copy run start

Step 2: Configure R2

Router(config)# hostname R2

R2(config)# interface GigabitEthernet0/1
R2(config-if)# ip address 10.0.0.2 255.255.255.252
R2(config-if)# no shutdown

R2(config)# interface GigabitEthernet0/0
R2(config-if)# ip address 192.168.2.1 255.255.255.0
R2(config-if)# no shutdown

R2# wr

Step 3: Add Static Routes

# On R1
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2

# On R2
R2(config)# ip route 192.168.1.0 255.255.255.0 10.0.0.1

Step 4: Test Connectivity

PC1> ping 192.168.2.10

84 bytes from 192.168.2.10 icmp_seq=1 ttl=62 time=5.234 ms
Success!

Useful Show Commands

show ip interface brief
show ip route
show running-config
show interfaces GigabitEthernet0/0

What I Learned

  • Configuration modes matter
  • Always save configs (wr)
  • Routing is bidirectional
  • Verify at each step

Hands-on practice beats theory every time.

Understanding VLANs and Trunking

VLANs let you segment networks logically without buying more switches.

What Problem Do VLANs Solve?

Without VLANs:

  • Everyone on same broadcast domain
  • All traffic visible to all devices
  • No logical separation
  • Need separate switches per department

With VLANs:

  • Logical network separation
  • Better security and performance
  • One switch, multiple networks

VLAN Types

Access Ports

  • Connect to end devices (PCs, phones)
  • Belong to ONE VLAN
  • Traffic is untagged

Trunk Ports

  • Connect switches together
  • Carry MULTIPLE VLANs
  • Traffic is tagged (802.1Q)

Basic VLAN Configuration

# Create VLANs
SW1(config)# vlan 10
SW1(config-vlan)# name SALES

SW1(config)# vlan 20
SW1(config-vlan)# name ENGINEERING

# Assign access port
SW1(config)# interface fa0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10

# Configure trunk
SW1(config)# interface fa0/24
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20

Verification Commands

show vlan brief
show interfaces trunk
show interfaces fa0/1 switchport

Inter-VLAN Routing

VLANs can't communicate without routing. Two options:

Router on a Stick

Router(config)# interface Gi0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

Layer 3 Switch

Switch(config)# ip routing
Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0

Best Practices

  • Don't use VLAN 1 for user traffic
  • Match native VLAN on both trunk ends
  • Use meaningful VLAN names
  • Document your VLAN scheme

Static vs Dynamic Routing: When to Use Each

Understanding when to manually configure routes versus letting protocols handle it.

Static Routing

What it is: Manually configured routes that don't change

Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2

Pros:

  • Simple and predictable
  • No routing protocol overhead
  • Complete control
  • Secure (no route advertisements)

Cons:

  • Doesn't adapt to failures
  • Manual updates required
  • Doesn't scale well

Best for: Small networks, stub networks, default routes

Dynamic Routing

What it is: Routers automatically share and learn routes

Common Protocols:

RIP (Routing Information Protocol)

  • Simple, distance-vector
  • Max 15 hops
  • Slow convergence
  • Good for: Learning only

OSPF (Open Shortest Path First)

  • Link-state protocol
  • Fast convergence
  • Scalable
  • Industry standard
  • Good for: Most enterprise networks

EIGRP (Enhanced Interior Gateway Routing Protocol)

  • Cisco proprietary (mostly)
  • Fast convergence
  • Easy to configure
  • Good for: Cisco-only networks

Comparison

Factor Static Dynamic
Configuration Manual Automatic
Adapts to changes No Yes
Bandwidth usage None Some (updates)
CPU/Memory Low Higher
Scalability Poor Good
Fault tolerance Poor Excellent

Real-World Scenarios

Use Static Routing When:

  • Network has < 5 routers
  • Topology rarely changes
  • Connecting to ISP (default route)
  • Security is critical

Use Dynamic Routing When:

  • Network has > 5 routers
  • Redundant paths exist
  • Topology changes frequently
  • Automatic failover needed

The OSI Model Explained Simply

The OSI Model: everyone mentions it, few explain it well. When I studied for CCNA, I memorized "Please Do Not Throw Sausage Pizza Away" and moved on. Now I'm actually understanding it.

What Is It?

The OSI (Open Systems Interconnection) Model describes how data travels through networks using 7 layers. Think of it as a postal system for digital information.

The 7 Layers (Bottom to Top)

Layer 1: Physical

Role: Physical transmission - cables, signals, bits

Examples: Ethernet cables, Wi-Fi radio waves, fiber optics

Devices: Cables, hubs, repeaters

Layer 2: Data Link

Role: Local network addressing with MAC addresses

Examples: Ethernet, Wi-Fi (802.11)

Devices: Switches, bridges

Layer 3: Network

Role: Routing between networks using IP addresses

Examples: IPv4, IPv6, ICMP (ping)

Devices: Routers

Layer 4: Transport

Role: Reliable delivery and error correction

Examples: TCP (reliable), UDP (fast)

Key concept: Ports (80=HTTP, 443=HTTPS)

Layer 5: Session

Role: Managing connections between applications

Example: Keeping you logged into a website

Layer 6: Presentation

Role: Data formatting, encryption, compression

Example: HTTPS encryption

Layer 7: Application

Role: User-facing network services

Examples: HTTP, FTP, SMTP, DNS, SSH

Why It Matters

The OSI Model is a troubleshooting framework:

  • No Wi-Fi? ‚Üí Layer 1 (Physical)
  • Can ping IP but not hostname? ‚Üí Layer 7 (DNS)
  • Can't reach other networks? ‚Üí Layer 3 (Routing)

Work through the layers systematically to find problems.
